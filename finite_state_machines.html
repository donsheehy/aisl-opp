Finite State Machines

Summary

A finite state machine (or finite automaton) is an abstract computational model with a fixed, finite memory. Part of Discrete Structures for Computation.

  • Deterministic and nondeterministic automata
    • A deterministic finite automaton (DFA) is a tuple $(Q, \Sigma, \delta, q_0, F)$ with transition function $\delta : Q \times \Sigma \to Q$.
    • A nondeterministic finite automaton (NFA) allows $\delta : Q \times \Sigma \to \mathcal{P}(Q)$ and often includes $\varepsilon$-transitions.
    • Every NFA has an equivalent DFA via the subset construction; both recognize exactly the regular languages.
  • Regular expressions and Kleene's theorem
    • Regular expressions are built from $\emptyset$, $\varepsilon$, and symbols $a \in \Sigma$ using union, concatenation, and Kleene star.
    • A language is recognized by some finite automaton if and only if it is described by a regular expression.

Every NFA has an equivalent DFA (subset construction); both recognize exactly the regular languages.

Kleene's theorem: a language is recognized by some finite automaton if and only if it is described by a regular expression.
